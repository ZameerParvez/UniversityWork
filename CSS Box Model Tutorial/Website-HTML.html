A Brief History of CSS

CSS stands for Cascading Style sheets.

It allows pages written in mark-up languages, such as HTML, to be styled. One of its main purposes is to separate presentation and content, it also has the advantage that many pages can be styled the same way by linking to the same style sheet, and another adventage which is that a single page can be styled by many style sheets (which is what makes it “cascading”).

This separation is known as separation of concerns and makes changing content or presentation much easier in the lifetime of a HTML document, and it also makes it easier to make the page more accessible, by having clean mark-up.

The newest CSS version is CSS 3 and is split into modules which add more functionality. CSS 3 is also a superset of CSS 2.1, so it is also backwards compatible.

CSS 2.1 was a revision over CSS 2, which fixed a few issues as well as removing and adding some features.

CSS 2 and CSS 1 allowed the authors of pages to style their own pages rather than the browsers forcing their style. At this point in time there were many competing methods of styling.

The Box Model

The Box Model is how html content is separated.

Every HTML tag creates a box which is made of a margin, border, padding and the content.

Illustration of the box model by W3C.
For more information click on the image.

The size of each of those values can be modified, to make the box bigger and smaller, and each side of the box can be modified separately, for example to change the size of the top margin you could use “margin-top: X px;”, where “X” is a positive number.

Some older browsers measure the sizes of boxes differently, for example IE8- included padding and border in the width property of boxes.

The DOM

The DOM is a W3C standard for accessing documents, which consists of 3 different parts, the Core, XML and HTML DOMs.

The HTML DOM is the structure of the html page, it stands for Document Object Model.

Illustration by Wikipedia.

When a page is loaded, the browser creates a tree of elements, which are all the html tags you use, these elements also have attributes, values and styles, which are all underneath their elements in the tree.

JavaScript can be used to access anything on the DOM tree, so you can change anything on the tree or add more elements to the tree with scripts. A method of accessing an element with JavaScript is Document.getElementById(“Elements ID”), there are also other methods such as finding an object by class name and tag name.

More detail about the DOM can be found here.

JavaScript is Evolving!

JavaScript is a programming language which is usually used for client side scripting, so that web pages can be more interactive. It has also been standardised in the ECMAScript language specification, most browsers support ECMAScript 5 and a few support the newer ECMAScript 6, which adds new features, such as being able to declare constants.

For a list of all added features to ECMAScript 6 click here.

This Video by Front End TV,
is an introduction to EcmaScript 6.

If you want to use the features in ECMAScript 6, to make your code more compatible, you can transpile your code to ECMAScript 5 using a tool called Babel, although there are a few features of ECMAScript 6 that cannot be used. A transpiler is software which compiles the source code of a language into the source code of another language, which will still perform the same task.

DIY Box Model

The files for the box model can be downloaded here.

The layers of the box are made by nesting “div” tags within each other, this is necessary because for demonstration purposes the layers of the box need to be coloured, which cannot usually be done to the margin.

There is also a key so that the matching colour tell you which layer is which, and so that the inputs can change their respective coloured boxes size. These inputs are also limited to prevent negative numbers and to keep the box model reasonably sized.

The inputs call functions in the JavaScript file, which modify the CSS to increase and decrease the padding sizes of the boxes which represent the layers of the box model.

The CSS for the box model first removes all margins from the layers, and sets the page’s margins to auto, so that all content is contained in the centre of the page, it also matches the colours of the boxes and the key. The padding is also what determines the size of the boxes, so the CSS file also initialises them to a size of 20px.

The JavaScript for this is ECMAScript 5 code which has been transpiled from ECMAScript 6, with Babel. The file also contains the original source code, which has been commented out, and if you compare it to the ECMAScript 5 code you can see a few differences, such as the use of “const” and the arrow functions in the original, which have advantages over functions and “var”, although it makes no difference here.
